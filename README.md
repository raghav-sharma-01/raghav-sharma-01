## About

I am a final-year Computer Science & Engineering (AI/ML) undergraduate at MIET, affiliated with Dr. A.P.J. Abdul Kalam Technical University, with a deep focus on building production-grade software that sits at the intersection of artificial intelligence and scalable full-stack engineering. My work spans intelligent backend systems, data-intensive ML pipelines, and enterprise-grade frontend interfaces — each built with the precision and rigour expected at senior engineering levels.

I approach software with a product-engineering mindset: understanding the problem domain before writing a single line of code, designing for extensibility from day one, and measuring success by user impact rather than technical novelty. My AI/ML work goes beyond model training — I focus on how to deploy, monitor, and iterate on ML systems in real-world production environments where reliability, latency, and cost all matter.

On the full-stack side, I build with the MERN ecosystem and design interfaces that reject generic aesthetics in favour of purposeful, information-dense UX — influenced by Bloomberg Terminal, Palantir Foundry, and institutional-grade tooling.

**Open To:** Campus placements (2027 batch) · AI/ML internships · Full-stack engineering roles · Research collaborations · Open source contribution

## Featured Projects

<details>
<summary><strong>🐄 CattleGPT — AI-Powered Livestock Health Intelligence Platform</strong></summary>

<br/>

CattleGPT is my final-year B.Tech capstone project — an end-to-end AI system for livestock disease diagnosis, health monitoring, and veterinary decision support. The platform integrates computer vision for visual symptom analysis, NLP for conversational diagnostics, and a retrieval-augmented generation pipeline to surface veterinary knowledge in real time. Designed for deployment in low-connectivity rural environments, CattleGPT bridges the gap between frontier AI capabilities and agricultural accessibility.

<br/>

| Attribute | Detail |
|-----------|--------|
| **Stack** | Python · FastAPI · React · MongoDB · TensorFlow · LangChain · Gemini API · FAISS |
| **Scale** | Multi-species disease taxonomy across 50+ documented cattle conditions |
| **Performance** | Sub-2s inference latency · 91%+ diagnostic accuracy on held-out validation set |
| **Security** | Role-based access · API key isolation · Environment-scoped credential management |
| **Impact** | Veterinary advisory tool targeting smallholder farmers across rural India |
| **Repository** | [github.com/raghav-sharma-01](https://github.com/raghav-sharma-01) |

The architecture follows a modular microservices pattern: an image ingestion service handles multi-frame symptom input, a classifier pipeline produces differential diagnoses, and a RAG agent queries a curated veterinary knowledge base to generate explainable, citation-backed responses. The frontend is built to function in bandwidth-constrained conditions, prioritising progressive loading and offline-first data caching.

</details>

<details>
<summary><strong>📊 Dark Data Analyzer — Enterprise Analytics Terminal</strong></summary>

<br/>

Dark Data Analyzer is a full SaaS analytics platform designed with an intentional Bloomberg/Palantir terminal aesthetic — rejecting the conventional "AI product" visual language in favour of high-information-density UX powered by IBM Plex Mono typography, acid-green accents on a near-black canvas, and a layout architecture that puts data first. The platform performs multi-dimensional analysis on uploaded datasets and surfaces statistical patterns, anomalies, and visualisations through a purpose-built terminal interface.

<br/>

| Attribute | Detail |
|-----------|--------|
| **Stack** | React · Node.js · Express · MongoDB · D3.js · Python (analysis engine) |
| **Scale** | Handles CSV/JSON datasets up to 500K rows with client-side chunked rendering |
| **Performance** | Analysis pipeline completes in under 3s on median dataset size |
| **Security** | Session-scoped data isolation · No persistent user data storage by default |
| **Impact** | Live production deployment on Netlify · Available at public URL |
| **Repository** | [github.com/raghav-sharma-01](https://github.com/raghav-sharma-01) |

The design philosophy is deliberately anti-generic: structural devices encode real information hierarchy, typography weights carry semantic meaning, and every animation serves a functional purpose rather than aesthetic novelty. The platform demonstrates that tooling built for engineers can be both technically rigorous and visually distinctive.

</details>

<details>
<summary><strong>🎙️ Telephonic Voice Agent — Real-Time Conversational AI</strong></summary>

<br/>

A production-grade telephonic voice agent integrating VideoSDK for real-time audio transport and Google Gemini for conversational AI inference. The system handles live audio streams, performs speech-to-text transcription, routes utterances through a Gemini-powered dialogue engine, and synthesises responses back into the call channel — achieving near-human conversational latency in a fully automated pipeline.

<br/>

| Attribute | Detail |
|-----------|--------|
| **Stack** | Python · VideoSDK · Gemini API · FastAPI · WebSocket · dotenv |
| **Scale** | Single-channel MVP with architecture designed for horizontal scaling |
| **Performance** | End-to-end voice round-trip latency under 1.2s on stable connection |
| **Security** | Credential isolation via `.env` · `.gitignore` best practices enforced |
| **Impact** | Proof-of-concept for automated telephonic customer support pipelines |
| **Repository** | [github.com/raghav-sharma-01/Telephonic-voice-agent](https://github.com/raghav-sharma-01) |

Built to demonstrate practical agentic AI integration — moving beyond chatbot UX into real-time multimodal voice pipelines. The project also serves as a reference implementation for secure API credential management in Python-based AI projects.

</details>

<details>
<summary><strong>🔗 DLVIZR — Data Linkage Visualizer (Entity Resolution Dashboard)</strong></summary>

<br/>

DLVIZR is an enterprise-grade entity resolution and data linkage visualisation dashboard inspired by Palantir Foundry's operational intelligence interface. Built as a single-file interactive HTML application, it features a live-updating canvas pipeline diagram, real-time log streaming, and a multi-panel control architecture that allows operators to configure, execute, and monitor record linkage workflows end-to-end.

<br/>

| Attribute | Detail |
|-----------|--------|
| **Stack** | Vanilla JS · HTML5 Canvas · CSS Grid · Custom SVG rendering engine |
| **Scale** | Simulates entity resolution across datasets with configurable cardinality |
| **Performance** | 60fps canvas rendering · Zero external dependencies · Sub-100KB bundle |
| **Security** | Fully client-side · No data egress · No external API calls |
| **Impact** | Portfolio flagship demonstrating systems thinking and UI engineering depth |
| **Repository** | [github.com/raghav-sharma-01](https://github.com/raghav-sharma-01) |

DLVIZR demonstrates that complex data engineering tooling can be packaged as a portable, dependency-free artifact without sacrificing visual or functional fidelity. The design language — dense information hierarchy, monochrome base with selective colour encoding, and structured panel layout — is a deliberate reference to institutional-grade data operations software.

</details>
